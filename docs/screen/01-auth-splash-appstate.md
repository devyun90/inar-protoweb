## 3. 상태 분기

### 기본 (정상 · 스플래시)
- 앱 상태 API 호출 중 로고 + 로딩 인디케이터 노출. 정상 응답 시 인증/홈 라우트로 자동 분기.

### 강제 업데이트 (force_update = true)
- 1-B 전체화면 모달. 닫기/뒤로가기 차단. CTA만 활성, 탭 시 스토어 딥링크.

### 권장 업데이트 (optional_update = true)
- 1-B와 유사하되 "나중에" 보조 버튼 추가 → 탭 시 통과(로그인/메인 진입).

### 서버 점검 (maintenance = true)
- 1-C 전체화면. "다시 시도" 탭 시 상태 API 재호출. 점검 지속 시 동일 화면 유지.

### 로딩 (스켈레톤)
- 스플래시 자체가 로딩 상태. 상태 API 응답 지연 시 로딩 인디케이터 유지 (타임아웃 10초).

### 에러 (네트워크/상태 API 실패)
- "연결할 수 없어요" + `arrow-clockwise` 재시도 버튼. 오프라인 미지원 안내.

## 4. 인터랙션

| 동작 | 대상 | 결과 |
|------|------|------|
| 자동 | 스플래시 | 상태 API 호출 → 분기 (스토어/점검/로그인/메인) |
| 탭 | 스토어로 이동 | OS 스토어 앱 딥링크 (iOS App Store / Android Play) |
| 탭 | 다시 시도 (점검) | 상태 API 재호출 |
| 탭 | 나중에 (권장 업데이트만) | 모달 닫고 다음 단계 진입 |
| 탭 | 재시도 (에러) | 상태 API 재호출 |
| 뒤로가기 | 강제 업데이트/점검 | 무효화 (시스템 백 차단) |

## 5. 권한별 차이

| 요소 | ADMIN (주양육자) | USER (보조양육자) |
|------|------------------|-------------------|
| 전체 | 해당 없음 (로그인 전 화면 · 권한 무관) | 해당 없음 |

> 본 화면은 인증 이전 단계로 역할(role_id)이 결정되지 않았다. 모든 사용자에게 동일하게 동작.

## 6. API 연동

### 6-1. 데이터 needs (구체적으로 — API 계약의 입력)

**읽기(Read):**
| 데이터 | 필드 | 타입 | 비고 |
|--------|------|------|------|
| 앱 상태 | platform | enum(ios/android) | 클라이언트 전송 |
| | app_version | string (semver) | 현재 설치 버전 |
| | build_number | int | 빌드 번호 |
| 상태 응답 | min_supported_version | string (semver) | 강제 업데이트 기준 |
| | latest_version | string (semver) | 최신 버전 |
| | force_update | bool | true → 차단 모달 |
| | optional_update | bool | true → 권장(나중에 가능) |
| | store_url | string (url) | 플랫폼별 스토어 링크 |
| | release_notes | string[] | 변경 요약 항목 |
| | maintenance | bool | true → 점검 화면 |
| | maintenance_start | datetime (KST) | 점검 시작 |
| | maintenance_end | datetime (KST) | 점검 종료 |
| | maintenance_message | string | 점검 안내 문구 |

**쓰기(Write):**
| 액션 | 필드 | 타입 | 검증 | 비고 |
|------|------|------|------|------|
| (없음) | — | — | — | 읽기 전용 상태 체크 |

## 7. 데이터·검증 규칙

| 입력 필드 | 필수 | 형식/범위 | 단위 | 에러 메시지 |
|-----------|------|-----------|------|-------------|
| app_version | Y | semver `x.y.z` | — | (클라이언트 자동 수집) |
| build_number | Y | 양의 정수 | — | — |
| 버전 비교 | — | `current < min_supported` → 강제 | — | — |

> 버전 비교는 semver 규칙. `current < min_supported_version` 이면 force_update 처리. store_url 미수신 시 플랫폼 기본 스토어 페이지로 폴백.
