## 3. 상태 분기

### 기본 (데이터 있음)
- 로고 + 태그라인 + 소셜 버튼 4종. iOS는 Apple 노출, Android는 Apple 숨김(정책상).

### 로딩 (소셜 인증 진행)
- 버튼 탭 시 전체 dim 오버레이 + 스피너. OAuth 시트(시스템) 노출 중 입력 차단.

### 에러 (인증 실패/취소)
- 사용자 취소: 무동작(스낵바 없이 복귀).
- 인증 실패: 하단 스낵바 "로그인에 실패했어요. 다시 시도해 주세요." + 버튼 재활성.
- 네트워크 오류: 스낵바 "네트워크 연결을 확인해 주세요."

### 분기 결과 (인증 성공 후)
- 최초 가입(is_new_user=true) → 약관 동의(최초) 화면.
- 기존 회원 + 미동의 약관 존재 → 약관 재동의 화면.
- 기존 회원 + 홈 없음 → 온보딩.
- 기존 회원 + 홈 있음 → 메인(서버 공지 체크 경유).

## 5. 권한별 차이

| 요소 | ADMIN (주양육자) | USER (보조양육자) |
|------|------------------|-------------------|
| 전체 | 해당 없음 (로그인 전 화면 · 권한 무관) | 해당 없음 |

> 역할은 로그인 후 온보딩 단계에서 결정된다. 홈 생성자는 자동 ADMIN, 초대 수락자는 USER. 본 화면에서는 권한 구분 없음.

## 6. API 연동

### 6-1. 데이터 needs (구체적으로 — API 계약의 입력)

**읽기(Read):**
| 데이터 | 필드 | 타입 | 비고 |
|--------|------|------|------|
| 인증 결과 | access_token | string (JWT) | 세션 토큰 |
| | refresh_token | string | 갱신 토큰 |
| | is_new_user | bool | true → 약관 동의 분기 |
| | user_id | string | 사용자 식별 |
| | email | string | 소셜 계정 이메일 |
| | has_pending_terms | bool | 미동의/변경 약관 존재 |
| | has_home | bool | 소속 홈 존재 여부 |

**쓰기(Write):**
| 액션 | 필드 | 타입 | 검증 | 비고 |
|------|------|------|------|------|
| 소셜 로그인 | provider | enum(google/apple/kakao/naver) | 필수 | |
| | id_token | string | provider 발급 토큰, 서명 검증 | 서버 검증 |
| | access_token | string | 카카오/네이버는 access_token 사용 | 서버에서 프로필 조회 |
| | device_info | object | 선택 | 푸시 토큰 등 |

### 6-2. 엔드포인트 매핑 (TBD — backend spec에서 확정)

|------|--------------|------|

| 토큰 갱신 | `TBD` (POST auth/refresh) | refresh_token |

## 7. 데이터·검증 규칙

| 입력 필드 | 필수 | 형식/범위 | 단위 | 에러 메시지 |
|-----------|------|-----------|------|-------------|
| provider | Y | google / apple / kakao / naver | — | — |
| id_token | Y (google/apple) | provider 발급 유효 토큰 | — | "로그인에 실패했어요. 다시 시도해 주세요." |
| access_token | Y (kakao/naver) | provider 발급 유효 토큰 | — | "로그인에 실패했어요. 다시 시도해 주세요." |
| 네트워크 | — | 온라인 필수(오프라인 미지원) | — | "네트워크 연결을 확인해 주세요." |

> id_token 검증·이메일 추출은 서버 책임. 클라이언트는 provider 시트 결과를 그대로 전달. 사용자 취소는 에러로 처리하지 않음.

## 8. 네비게이션

| 구분 | 경로 |
|------|------|
| 진입 | 스플래시(정상) → 미인증 시 / 세션 만료(401) 시 |
| 이탈 | 인증 성공 → 약관 동의 / 재동의 / 온보딩 / 메인 |
| 딥링크 | 초대 링크 진입 시 로그인 후 온보딩 초대 수락으로 연결 (state 보존) |
| 뒤로가기 동작 | 루트 화면 → 앱 종료 |


## 10. 접근성

| 항목 | 기준 |
|------|------|
| 터치 타깃 | 소셜 버튼 높이 ≥ 56px, 약관 링크 ≥ 48px |
