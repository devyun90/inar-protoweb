## 3. 상태 분기

### 기본 (체크 전)
- 필수 3 + 선택 2 모두 미체크. CTA 비활성(dim).

### 필수 충족
- 필수 3종 체크 완료 → CTA 활성. 선택 동의는 무관.

### 전체 동의 체크
- "전체 동의" 체크 → 필수+선택 일괄 체크. 하나라도 해제 시 "전체 동의" 자동 해제.

### 로딩 (제출 중)
- CTA 탭 후 스피너. 동의 기록 저장 API 응답 대기.

### 에러 (저장 실패)
- 스낵바 "약관 동의 저장에 실패했어요. 다시 시도해 주세요." + CTA 재활성.

## 7. 데이터·검증 규칙

| 입력 필드 | 필수 | 형식/범위 | 단위 | 에러 메시지 |
|-----------|------|-----------|------|-------------|
| 서비스 이용약관 | Y | true 필수 | — | "필수 약관에 동의해 주세요" |
| 개인정보 수집·이용 | Y | true 필수 | — | "필수 약관에 동의해 주세요" |
| 개인정보 제3자 제공 | Y | true 필수 | — | "필수 약관에 동의해 주세요" |
| 마케팅 정보 수신 | N | true/false | — | — |
| 푸시 알림 수신 | N | true/false | — | — |

> 필수 3종 전부 true여야 CTA 활성. 선택 2종은 기본 false, 사용자가 명시 동의 시 true. 동의 시점의 약관 version을 함께 기록.

## 9. 데이터 예시

```

  [□] 전체 동의 (선택 포함)
  [□] (필수) 서비스 이용약관
  [□] (필수) 개인정보 수집 및 이용
  [□] (필수) 개인정보 제3자 제공
  [□] (선택) 마케팅 정보 수신
  [□] (선택) 푸시 알림 수신
```

### 9-2. 더미 데이터 (정상 표시)
```
(필수 3종 + 마케팅 체크 상태)
  [□] 전체 동의 (선택 포함)   ← 푸시 미체크라 해제 상태
  [■] (필수) 서비스 이용약관          v1.2
  [■] (필수) 개인정보 수집 및 이용    v1.1
  [■] (필수) 개인정보 제3자 제공      v1.0
  [■] (선택) 마케팅 정보 수신         v1.0
  [□] (선택) 푸시 알림 수신           v1.0
  [동의하고 계속하기]  ← 활성
```
