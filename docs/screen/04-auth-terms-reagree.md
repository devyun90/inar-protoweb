## 3. 상태 분기

### 기본 (변경 약관 존재)
- 변경된 약관만 목록 표시. 변경 없는 약관은 미표시. 뒤로가기/닫기 차단(메인 진입 게이트).

### 필수 변경 미동의
- 변경 필수 약관 중 하나라도 미체크 → CTA 비활성.

### 선택만 변경
- 선택 약관만 변경된 경우에도 화면 노출. 단, 필수 변경분 없으면 CTA는 즉시 활성(선택은 동의 강제 아님).

### 로딩 (제출 중)
- CTA 탭 후 스피너. 재동의 저장 API 대기.

### 에러 (저장 실패)
- 스낵바 "동의 저장에 실패했어요. 다시 시도해 주세요." + CTA 재활성.

## 4. 인터랙션

| 동작 | 대상 | 결과 |
|------|------|------|
| 탭 | 변경 약관 전체 동의 | 표시된 변경 항목 일괄 토글 |
| 탭 | 개별 체크박스 | 토글, 필수 변경분 전부 시 CTA 활성 |
| 탭 | `[>]` (상세) | 변경 약관 상세 시트 (변경점 강조) |
| 탭 | 동의하고 계속하기 | 재동의 저장 → 서버 공지 체크/메인 |
| 탭 | 동의하지 않고 나가기 | 거부 다이얼로그 |
| 탭 | 다이얼로그-로그아웃 | 세션 종료 → 로그인 (다음 로그인 시 재노출) |
| 탭 | 다이얼로그-회원 탈퇴 | 탈퇴 플로우(시나리오 15) |
| 뒤로가기 | 화면 | 차단 (시스템 백 무효, 거부 다이얼로그 유도 안 함) |

## 5. 권한별 차이

| 요소 | ADMIN (주양육자) | USER (보조양육자) |
|------|------------------|-------------------|
| 재동의 화면 노출 | 변경 약관 있으면 노출 (동일) | 변경 약관 있으면 노출 (동일) |
| 거부 시 탈퇴 영향 | ADMIN 단독 홈이면 탈퇴 시 홈 삭제 경고 (시나리오 15 연계) | 탈퇴 시 홈에서만 이탈 |

> 약관 재동의 자체는 역할 무관 공통. 단, 거부→탈퇴 분기에서 ADMIN 단독 홈은 홈·아이 데이터 전체 삭제 경고가 추가된다.

## 6. API 연동

### 6-1. 데이터 needs (구체적으로 — API 계약의 입력)

**읽기(Read):**
| 데이터 | 필드 | 타입 | 비고 |
|--------|------|------|------|
| 변경 약관 목록 | terms_id | string | 변경된 약관만 |
| | type | enum(service/privacy_collect/privacy_third_party/marketing/push) | |
| | required | bool | 필수 여부 |
| | title | string | 표시 라벨 |
| | prev_version | string | 사용자 기동의 버전 |
| | new_version | string | 신규 버전 |
| | content_url | string (url) | 변경 본문 |
| | changed | bool | true 항목만 수신 |

**쓰기(Write):**
| 액션 | 필드 | 타입 | 검증 | 비고 |
|------|------|------|------|------|
| 재동의 제출 | agreements[].terms_id | string | 변경 필수분 포함 | |
| | agreements[].version | string | new_version | |
| | agreements[].agreed | bool | 변경 필수 = true | |
| 거부(로그아웃) | (세션 종료) | — | — | 동의 미저장 |
| 거부(탈퇴) | reason | string | 선택 | 탈퇴 플로우 위임 |

## 7. 데이터·검증 규칙

| 입력 필드 | 필수 | 형식/범위 | 단위 | 에러 메시지 |
|-----------|------|-----------|------|-------------|
| 변경 필수 약관 | Y | true 필수 | — | "변경된 필수 약관에 동의해 주세요" |
| 변경 선택 약관 | N | true/false | — | — |
| 게이트 통과 조건 | — | 변경 필수분 전부 동의 | — | — |

> 변경 약관이 없으면 본 화면을 거치지 않고 바로 다음 단계로 통과. 선택 약관만 변경 시 동의 없이도 통과 허용(선택 항목은 강제 아님).

## 8. 네비게이션

| 구분 | 경로 |
|------|------|
| 진입 | 로그인 성공 + has_pending_terms=true (기존 회원) |
| 이탈 | 동의 완료 → 서버 공지 체크 → 메인 / 거부 → 로그인 or 탈퇴 |
| 딥링크 | 변경 약관 상세 content_url |
| 뒤로가기 동작 | 차단 (메인 진입 게이트, 시스템 백 무효) |

## 9. 데이터 예시

### 9-1. 초기 상태 (데이터 없을 때)
```
(변경 약관 없음 — 화면 자체 미진입)
  → 게이트 통과, 다음 단계(서버 공지/메인)로 자동 이동
```

### 9-2. 더미 데이터 (정상 표시)
```
(필수 1 + 선택 1 변경)
  [□] 변경 약관 전체 동의
  [□] (필수) 개인정보 제3자 제공   v1.0 → v2.0  [변경됨]
  [□] (선택) 마케팅 정보 수신       v1.0 → v1.1  [변경됨]
  ※ 변경되지 않은 약관은 표시되지 않아요
  [동의하고 계속하기]  ← 필수 미체크라 비활성
  [동의하지 않고 나가기]
```
