## 3. 상태 분기

### 진입 불가형 (type=blocking)
- 1-A 전체화면. 뒤로가기/닫기 차단. "다시 시도"만 활성 → 공지 상태 재조회. 해제 전까지 메인 진입 불가.

### 팝업형 (type=popup)
- 1-B 다이얼로그. 메인 위에 오버레이. "확인"으로만 닫힘. 배경 탭으로는 닫히지 않음.

### 팝업 — 7일 숨김 체크
- 체크 후 "확인" → 해당 공지 id를 7일간 로컬 suppress. 7일 내 동일 공지 미표시.

### 공지 없음
- 게이트 통과, 화면 미표시 → 메인 직행.

### 로딩
- 공지 상태 조회 중. 게이트 단계라 스플래시/메인 로딩에 흡수.

### 에러 (조회 실패)
- 진입 불가형 판단 불가 시 안전하게 차단하지 않고 메인 진입 허용(공지는 best-effort). 단 blocking 캐시가 있으면 유지.

## 4. 인터랙션

| 동작 | 대상 | 결과 |
|------|------|------|
| 탭 | 다시 시도 (진입 불가) | 공지 상태 재조회, 해제 시 메인 진입 |
| 탭 | 7일간 다시 보지 않기 | 체크 토글 (로컬 suppress 예약) |

| 탭 | 배경 dim (팝업) | 무동작 (확인으로만 닫힘) |
| 뒤로가기 | 진입 불가형 | 차단 (시스템 백 무효) |
| 뒤로가기 | 팝업형 | 팝업 닫힘 (suppress 미적용) |

## 5. 권한별 차이

| 요소 | ADMIN (주양육자) | USER (보조양육자) |
|------|------------------|-------------------|
| 공지 표시 | 동일 (전 사용자 대상) | 동일 |
| 공지 작성/관리 | 해당 없음 (운영 관리자 영역, 앱 내 미제공) | 해당 없음 |

> 본 화면의 서버 공지는 운영 측에서 발행하는 시스템 공지로, 앱 내 역할(ADMIN/USER)과 무관하게 동일 표시된다. (홈 내 ADMIN 작성 "홈 공지"는 별도 화면 `home-todo-notice-detail`에서 다룸.)

## 6. API 연동

### 6-1. 데이터 needs (구체적으로 — API 계약의 입력)

**읽기(Read):**
| 데이터 | 필드 | 타입 | 비고 |
|--------|------|------|------|
| 공지 | notice_id | string | 공지 식별 (suppress 키) |
| | type | enum(blocking/popup) | 표시 방식 |
| | title | string | 제목 |
| | body | string (markdown/plain) | 본문 |
| | impact_scope | string | 영향 범위 (blocking) |
| | recover_eta | datetime | 예상 복구 (blocking) |
| | detail_url | string (url) | 자세히 보기 (popup) |
| | starts_at | datetime | 노출 시작 |
| | ends_at | datetime | 노출 종료 |
| | dismissible | bool | 닫기 가능 여부 |
| | resolved | bool | blocking 해제 여부 |

**쓰기(Write):**
| 액션 | 필드 | 타입 | 검증 | 비고 |
|------|------|------|------|------|
| 7일 숨김 | notice_id | string | 로컬 저장 | 클라이언트 보관 |
| | suppress_until | datetime | now + 7일 | 로컬 |

### 6-2. 엔드포인트 매핑 (TBD — backend spec에서 확정)

| 액션 | METHOD /path | 비고 |
|------|--------------|------|
| 활성 공지 조회 | `TBD` (GET notices/active) | type·resolved 반환 |
| blocking 상태 재확인 | `TBD` (GET notices/active) | "다시 시도" 시 재호출 |
| 7일 숨김 | (로컬 저장) | 서버 호출 없음, 디바이스 로컬 storage |

## 7. 데이터·검증 규칙

| 입력 필드 | 필수 | 형식/범위 | 단위 | 에러 메시지 |
|-----------|------|-----------|------|-------------|
| type | Y | blocking / popup | — | (서버 제공) |
| 7일 숨김 체크 | N | bool (popup만) | — | — |
| suppress_until | — | now + 7일 | 일 | — |
| blocking 해제 | — | resolved=true 시 통과 | — | — |

> blocking 공지는 `resolved=true` 또는 `ends_at` 경과 전까지 메인 진입을 막는다. popup은 노출 1회 후 "확인"으로 닫히며, 7일 숨김 체크 시 `notice_id`별로 로컬에 suppress_until 저장. suppress 만료 후 재노출.

## 10. 접근성

| 항목 | 기준 |
|------|------|
| 터치 타깃 | 버튼·체크 ≥ 48px |
| 색 대비 | 본문 `#4f453f` on `#fcf9f7`/`#ffffff` ≥ 4.5:1 |
| 스크린리더 라벨 | "서비스 이용 안내, 진입 제한 공지. 다시 시도 버튼" / "7일간 다시 보지 않기 체크박스" |
| 포커스 순서 | 아이콘(데코) → 제목 → 본문 → (팝업: 7일 숨김) → 확인/다시 시도 |
