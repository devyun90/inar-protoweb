## 3. 상태 분기

### 기본 (입력 전)
- 필수(이름·생년월일·성별) 미입력. "저장" 비활성. 사진/키/몸무게/메모는 선택.
- "아이 추가" 버튼은 저장 전에는 비활성 또는 미표시.

### 입력 충족
- 필수 3종 충족 → "저장" 활성.

### 저장 완료 → 아이 추가 가능
- 저장 성공 후 "아이 추가" 버튼 활성. 탭 시 폼 초기화 + 생년월일 복사(쌍둥이 편의).

### 로딩 (저장 중)
- 아이 프로필 생성 API 호출 중 스피너.

### 에러 (저장 실패)
- 스낵바 "프로필 저장에 실패했어요. 다시 시도해 주세요." + 재활성.

### 다자녀 (아이 추가 후)
- 상단에 등록된 아이 칩/태그 표시 (예: "하은 ✓"). 추가 등록 폼 진행.

## 4. 인터랙션

| 동작 | 대상 | 결과 |
|------|------|------|
| 탭 | 프로필 사진 | 사진 선택 시트 (카메라/갤러리) — 선택 |

## 5. 권한별 차이

| 요소 | ADMIN (주양육자) | USER (보조양육자) |
|------|------------------|-------------------|
| 본 화면 진입 | 가능 (홈 생성자, 온보딩 중 ADMIN 자동 부여) | 해당 없음 (초대 수락자는 본 화면 미진입, 메인 직행) |
| 아이 등록 | 가능 | — (홈에 등록된 아이를 상속) |
| 아이 추가 | 가능 (다자녀/쌍둥이) | — |

> 본 화면은 홈을 새로 만든 사용자(자동 ADMIN) 전용 단계다. 초대로 합류한 USER는 온보딩에서 아이 프로필 등록을 거치지 않으며, 아이 추가/수정은 ADMIN 권한 영역이다.

## 9. 시나리오 변경: 젖병/분유 등록 분리

### 변경 사유
- 아이 등록 직후 젖병/분유 설정까지 진행하면 **온보딩 피로감** 증가
- 사용자가 아이 등록만으로도 즉시 서비스 이용 가능해야 함
- 기기(분유포트)를 보유한 사용자에게는 **기기 등록을 우선 유도**하는 것이 자연스러움

### 기기 등록 우선 유도 → 젖병/분유 자연 연결 전략

| 시점 | 유도 방식 | 상세 |
|------|-----------|------|
| **온보딩 완료 직후** | Alert (1회) | "분유포트를 가지고 계신가요? 기기를 등록하면 자동 수유 기록이 시작돼요" + [등록하기] / [나중에] |
| **기기 등록 완료 직후** | 연속 진행 안내 | "젖병과 분유를 등록하면 자동 출수가 가능해요" → 젖병/분유 설정으로 자연 이동 |
| **자동 출수 기능 사용 시** | 리다이렉트 (강제) | 젖병 미등록 상태에서 자동 출수 → 젖병/분유 설정 화면으로 강제 이동 + "이 기능을 사용하려면 젖병 등록이 필요해요" |
| **가입 D+1 (기기 미등록)** | 푸시 알림 (로컬) | "분유포트를 등록하면 더 편리하게 기록할 수 있어요" |

### 연속 흐름 (빠르게 한 번에 끝내기)

```
[Alert] 등록하기
  → 기기 등록 (AP → Wi-Fi)
  → 완료 → "젖병/분유도 등록할까요?"
     → [바로 등록] → 젖병 캘리브레이션 + 분유 등록
     → [나중에] → 대시보드 (설정에서 재진입 가능)
```

### 연결 화면 변경 (Before → After)

```
[Before]
온보딩 → 아이 프로필 → 젖병/분유 설정(선택) → 메인

[After]
온보딩 → 아이 프로필 → 메인 (즉시)
                              └→ Alert: 기기 등록 유도
                                   └→ 기기 등록 → 젖병/분유 연속 진행
```
