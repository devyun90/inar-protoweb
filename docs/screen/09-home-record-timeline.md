## 3. 상태 분기

### 기본 (데이터 있음)
- 시간대별(새벽/오전/오후/저녁) 그룹핑, 최신순 정렬
- 각 기록 카드: 카테고리 아이콘 + 제목 + 시간 + 작성자, 메모 있으면 인디케이터

### 빈 상태 (해당 날짜 기록 없음)
- `baby` 아이콘 + "이 날의 기록이 없어요" + "기록 추가하기" CTA (FAB 강조)

### 로딩 (스켈레톤)
- 시간대 헤더 1 + 카드 3행 shimmer (radius 16px, h 72px)

### 에러
- 전체: 에러 아이콘 + "기록을 불러올 수 없어요" + 재시도
- 부분(특정 그룹): 해당 그룹만 재시도

### 퀵등록 — 초기 사용자 (커스텀 미설정)
- 기본값 고정: 모유 → 분유 → 수면 → 기저귀 (항상 이 4개, 빈도 기반 아님)
- 입력값 모드 기본: 모유="매번 입력", 분유="매번 입력", 수면="매번 입력", 기저귀="매번 입력"
- 편집 버튼(✏) 항상 노출

### 퀵등록 — 커스텀 설정 완료
- 사용자가 선택한 카테고리·순서대로 표시 (1~8개, 수동입력 포함 가능)
- 각 항목의 입력값 모드(고정값/최근값/매번입력)에 따라 동작 분기:
  - **고정값**: 탭 즉시 저장 (사전 설정한 값으로 기록 생성, 확인 토스트)
  - **최근값**: 마지막 기록값을 프리필 → 확인 탭 1회로 저장
  - **매번 입력**: 기존처럼 입력 바텀시트 오픈

### 퀵등록 — 수동입력 항목
- 사용자가 직접 정의한 커스텀 기록 유형 (예: "약 복용", "물 50ml", "비타민D")
- 탭 시 고정 메모(설정한 경우)와 함께 즉시 기록 생성
- 수동입력은 activity category="custom", custom_label 필드로 저장

## 6. API 연동

### 6-1. 데이터 needs

**읽기(Read):**
| 데이터 | 필드 | 타입 | 비고 |
|--------|------|------|------|
| 기록 목록 | baby_id | string | 활성 아이 |
| | date (From~To) | date | 선택 날짜 1일 |
| | category | enum | 필터 시 |
| 기록 항목 | activity_id | string | |
| | category | enum(breast_feeding/formula_feeding/sleep/diaper/growth/hospital/vaccine) | |
| | title/value | string/number | 카테고리별 |
| | recorded_at | datetime | 정렬·그룹 기준 |
| | author(name, user_id) | object | 작성자 |
| | has_memo | bool | 인디케이터 |
| 퀵등록 설정 | user_id | string | 사용자별 설정 |
| | quick_items | object[] | [{category, order, value_mode, preset_value}] |
| | custom_items | object[] | [{custom_label, icon, fixed_memo, order}] |
| | is_custom | bool | true=커스텀, false=기본값(수유/수면/기저귀) |
| 최근 기록값 | baby_id+category | object | 마지막 기록의 입력값 (최근값 모드용) |

**쓰기(Write):**
| 액션 | 필드 | 타입 | 검증 | 비고 |
|------|------|------|------|------|
| 삭제 | activity_id | string | ADMIN만 | 스와이프 |

| | custom_items | object[] | 0~N개 | 수동입력 항목 |
| 퀵등록 초기화 | — | — | — | is_custom=false → 기본 3개 복귀 |
| 고정값 즉시 기록 | category, preset_value | varies | 카테고리 규칙 준수 | 확인 없이 저장 |

## 11. 퀵등록 커스터마이징 정책

### 기본값 (커스텀 미설정 시 — 모든 사용자 동일)

| 항목 | 값 |
|------|-----|
| 표시 칩 | 모유 → 분유 → 수면 → 기저귀 (고정 4개) |
| 입력값 모드 | 전부 "매번 입력" |
| 수동입력 | 없음 |

> 빈도 기반 자동 정렬 없음. 기본값은 항상 수유/수면/기저귀 고정.

### 커스텀 규칙

| 항목 | 제약 |
|------|------|
| 최소 표시 개수 | 1개 |
| 최대 표시 개수 | 8개 (카테고리 6 + 수동입력 N, 합계 최대 8) |
| 순서 | 드래그로 자유 변경 |
| 입력값 모드 | 카테고리별 개별 설정 가능 |
| 수동입력 항목 | 사용자가 직접 추가 (이름+아이콘+고정메모) |
| 저장 | 사용자(user_id)별 — 같은 홈이라도 구성원마다 다름 |
| 기기 동기화 | 서버 저장 (다른 기기에서도 동일 설정 유지) |
| 기본값 리셋 | "기본값으로 되돌리기" → 모유/분유/수면/기저귀 + 매번입력, 수동입력 삭제 |

### 입력값 모드 상세

| 모드 | 동작 | 칩 표시 | UX |
|------|------|---------|-----|
| **고정값 사용** | 탭 즉시 기록 생성 (사전 설정한 값) | 칩에 값 표시 (예: "🍼120ml") | 0탭 (탭=완료) |
| **최근값 사용** | 마지막 기록값을 프리필 → 확인 1탭 | 칩에 "🍼최근" 표시 | 1탭 (확인) |
| **매번 입력** | 입력 바텀시트 풀 오픈 | 칩에 라벨만 (예: "🍼수유") | N탭 (입력 후 저장) |

### 고정값 설정 예시 (카테고리별)

| 카테고리 | 고정값 설정 항목 | 예시 |
|----------|------------------|------|
| 모유 | 방향 + 시간 | 양쪽 15분 |
| 분유 | 양 (+ 분유종류) | 120ml |
| 수면 | 시작/종료 | 수면 시작 (현재 시각) |
| 기저귀 | 유형 | 소변 |
| 성장 | 키 + 몸무게 | — (고정 부적합, "매번 입력" 권장) |
| 병원 | 병원명 + 메모 | — (고정 부적합) |
| 예방접종 | 접종명 | — (고정 부적합) |

### 수동입력 항목

| 항목 | 설명 |
|------|------|
| 용도 | 기존 카테고리에 없는 반복 행동 빠르게 기록 |
| 예시 | "약 복용", "모유수유 왼쪽 15분", "물 50ml", "비타민D" |
| 구성 | 항목 이름 (필수) + 아이콘 (기본 ✍️) + 고정 메모 (선택) |
| 동작 | 탭 시 즉시 기록 생성 (custom category, label+memo 저장) |
| 저장 | activity.category = "custom", activity.custom_label = 항목 이름 |

### 편집 접근 방법

| 방법 | 동작 |
|------|------|
| ✏ 편집 버튼 탭 | 퀵등록 영역 끝에 항상 노출 |
| 퀵등록 영역 롱프레스 | 편집 시트 오픈 (편집 버튼과 동일) |

### FAB와의 관계

| | 퀵등록 칩 | FAB |
|---|---|---|
| 역할 | 고정값/최근값으로 **초고속** 기록 | **전체** 카테고리 접근 (풀 입력) |
| 단계 | 0~1탭 (고정값/최근값) / N탭 (매번입력) | 2단계+ (FAB → 카테고리 → 입력) |
| 커스텀 | 표시 항목·순서·입력값모드·수동입력 | 변경 불가 (항상 전체 표시) |
| 위치 | 날짜 하단 고정 | 우하단 플로팅 |
