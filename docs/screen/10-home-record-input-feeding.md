## 3. 상태 분기

### 기본 (모유 입력)
- 방향 기본값: 직전 모유 기록의 방향 또는 "양쪽"
- 시간 기본값: 직전 모유 기록의 시간 또는 15분
- 시간(recorded_at): now

### 기본 (분유 입력)
- 수유량 기본값: 직전 분유 기록의 양 또는 120ml
- 분유 종류: ACTIVE 분유 자동 선택 (미등록 시 필드 미표시)
- 시간(recorded_at): now

### 저장 가능 조건
- 모유: 방향 + 수유시간(1~120분) 필수
- 분유: 수유량(1~500ml) 필수

### 로딩 (저장 중)
- Optimistic UI: 저장 즉시 시트 닫고 타임라인 상단에 임시 카드 삽입(반투명)
- API 성공 시 임시 카드 확정, 실패 시 롤백 + 토스트

### 에러
- 검증 실패: 해당 입력 하단 인라인 에러 (저장 차단)
- 저장 실패(네트워크/5xx): "저장에 실패했어요" 토스트 + 임시 카드 롤백 + 재시도

## 5. 권한별 차이

| 요소 | ADMIN (주양육자) | USER (보조양육자) |
|------|------------------|-------------------|
| 모유/분유 기록 생성 | 가능 | 가능 |
| 작성자 표기 | 본인 user_id | 본인 user_id |
| 저장 후 수정 | 전체 기록 수정 가능 | 본인 작성 기록만 수정 |
| 저장 후 삭제 | 가능 | 불가 (정책상 USER 삭제 금지) |

> 입력(생성) 자체는 권한 차이 없음. 차이는 저장 이후 [기록 상세]의 수정/삭제 권한에서 발생.

## 11. 변경 이력

### v3.3 — 이유식 제거, 모유/분유 분리

| Before | After |
|--------|-------|
| 수유 유형 세그먼트(모유/분유/이유식) 1화면 | **모유 입력** / **분유 입력** 각각 별도 화면 |
| FAB 카테고리: 수유 1개 | FAB 카테고리: **모유 + 분유** 2개 |
| category = "feeding" + feeding_type | category = "**breast_feeding**" / "**formula_feeding**" |
| 이유식(solid) 포함 | **이유식 제거** (MVP 범위 축소) |

**변경 사유:**
- 모유와 분유는 입력 필드가 완전히 다르므로 분리가 자연스러움
- 이유식은 이유식 시작 시기(보통 6개월+)부터 필요 — MVP 타겟(신생아~초기 영아)에 불필요
- 퀵등록 칩에서도 모유/분유를 각각 1탭으로 접근 가능해져 UX 개선
