## 3. 상태 분기

### 기본 (데이터 있음)
- 카테고리 헤더 + 상세 정보 카드 + 카테고리별 상세 영역 표시
- 수정 이력 있으면 "마지막 수정" 행 노출, 없으면 숨김

### 빈 상태 (데이터 없음 — 삭제됨/접근 불가)
- "삭제되었거나 찾을 수 없는 기록이에요" + [목록으로] 버튼
- (타 사용자가 삭제한 직후 진입 시) 동일 처리

### 로딩 (스켈레톤)
- 헤더 카드 + 정보 카드 2장 shimmer (radius 20px)

### 에러
- 전체: 에러 아이콘 + "기록을 불러올 수 없어요" + 재시도
- 삭제 실패: "삭제에 실패했어요" 토스트 + 재시도

## 5. 권한별 차이

| 요소 | ADMIN (주양육자) | USER (보조양육자) |
|------|------------------|-------------------|
| 상세 조회 | 가능 | 가능 (동일) |
| 수정 버튼/메뉴 | 모든 기록 노출 | 본인 작성 기록만 노출 (타인 작성은 읽기 전용) |
| 삭제 버튼/메뉴 | 노출 | 숨김 (본인 작성도 삭제 불가, 정책상 USER 삭제 금지) |
| 타인 작성 기록 | 수정·삭제 가능 | 읽기 전용 (수정/삭제 모두 숨김) |

> 권한 매트릭스(user_scenario 2-2): USER는 타인 작성 수정 불가, 전 항목 삭제 불가(API 403). UI에서 버튼 자체를 숨겨 시도 자체를 차단.

## 6. API 연동

### 6-1. 데이터 needs (구체적으로 — API 계약의 입력)

**읽기(Read):**
| 데이터 | 필드 | 타입 | 비고 |
|--------|------|------|------|
| 기록 상세 | activity_id | string | 필수 |
| | category | enum | 표시 분기 |
| | detail | object | 카테고리별 필드(amount_ml, duration_min, breast_side, diaper_type, stool_state, height_cm, weight_kg, head_cm, hospital_name, visit_reason, diagnosis, vaccine_code, dose_no, clinic_name, start_at, end_at 등) |
| | recorded_at | datetime | |
| | memo | string | nullable |
| | author | object(user_id,name) | 작성자 |
| | updated_at / updated_by | datetime/object | 수정 이력, nullable |
| 권한 판정 | current_user_id | string | 본인 작성 여부 비교 |
| | current_role | enum(ADMIN/USER) | 버튼 노출 분기 |

**쓰기(Write):**
| 액션 | 필드 | 타입 | 검증 | 비고 |
|------|------|------|------|------|
| 삭제 | activity_id | string | ADMIN만 | 확인 다이얼로그 후 |
| (수정 진입) | — | — | ADMIN 전체/USER 본인 | 실제 수정은 입력 화면에서 PATCH |

## 10. 접근성

| 항목 | 기준 |
|------|------|
| 터치 타깃 | 수정/삭제 버튼 48px, 더보기 48px |
| 색 대비 | 본문 #4f453f on #ffffff ≥ 4.5:1, 삭제(Error #ba1a1a) 대비 유지 |

| 포커스 순서 | 뒤로 → 헤더 → 상세 정보 → 카테고리 상세 → 수정 → 삭제 |
