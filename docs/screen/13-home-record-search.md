## 3. 상태 분기

### 기본 (검색 결과 있음)
- 결과는 날짜(내림차순) 그룹핑 → 각 그룹 내 시간 내림차순
- 키워드 매칭 텍스트는 부제에 노출, 매칭 구간 강조(굵게/하이라이트)
- 결과 요약 바에 총 건수 표시

### 빈 상태 (검색 결과 없음)
- `magnifying-glass` 아이콘 + "검색 결과가 없어요" + 보조 카피
- "필터 초기화" 버튼 → 키워드/기간/카테고리 전체 리셋 후 기본 목록 복귀
- 상단에 적용된 필터 요약 칩 표시 (어떤 조건으로 0건인지 인지)

### 초기 (검색 미수행)
- 입력 비어 있음, 카테고리 "전체" 선택, 기간 기본값(최근 7일)
- 결과 영역: "검색어나 필터를 선택해보세요" 안내 (placeholder)

### 로딩 (스켈레톤)
- 결과 요약 바 1줄 + 날짜 헤더 1 + 카드 3행 shimmer (radius 16px, h 72px)

### 에러
- 전체: 에러 아이콘 + "검색에 실패했어요" + 재시도
- 부분: 페이지네이션 추가 로드 실패 시 하단 "다시 시도" 인라인

## 4. 인터랙션

| 동작 | 대상 | 결과 |
|------|------|------|
| 탭 | 검색 입력 | 키보드 활성, 디바운스 300ms 후 검색 실행 |
| 입력 | 키워드 | 디바운스 후 결과 refetch |
| 탭 | clear(✕) | 키워드만 초기화, 결과 재조회 |
| 탭 | From/To 칩 | 캘린더 피커(미래일 불가, From≤To) |
| 탭 | 카테고리 칩 | 다중 토글 ("전체"는 단독 토글) |
| 탭 | 초기화 | 키워드+기간+카테고리 전체 리셋 |
| 탭 | 결과 카드 | 기록 상세 화면 |
| 탭 | 필터 초기화 CTA | Empty에서 전체 리셋 후 기본 목록 |
| 스크롤 | 결과 리스트 | 하단 도달 시 다음 페이지 로드 |
| Pull-to-refresh | 결과 영역 | 현재 필터로 재조회 |

## 5. 권한별 차이

| 요소 | ADMIN (주양육자) | USER (보조양육자) |
|------|------------------|-------------------|
| 검색·필터 사용 | 가능 | 가능 (동일) |
| 결과 조회 범위 | 홈 전체 기록 | 홈 전체 기록 (동일) |
| 결과 → 상세 진입 | 가능 | 가능 (수정/삭제 권한은 상세 화면에서 분기) |

> 검색 화면 자체는 조회 전용으로 권한 차이가 없다. 수정/삭제 권한은 진입한 [기록 상세] 화면에서 분기.
