## 3. 상태 분기

### 기본 (데이터 있음)
- 진행 중: 마감 기준 그룹핑(지남/오늘/예정), 마감 임박 상단
- 완료: 완료일 기준 내림차순, 날짜 필터 적용, 제목 취소선/흐림 처리

### 빈 상태 (데이터 없음)
- 진행 중: `list-checks` 아이콘 + "할 일을 추가해보세요" + [TODO 추가] CTA
- 완료: `check-circle` 아이콘 + "완료한 할 일이 없어요" (필터 적용 시 "해당 기간 완료 항목이 없어요")

### 로딩 (스켈레톤)
- 그룹 헤더 1 + 카드 3행 shimmer (radius 16px, h 72px)

### 에러
- 전체: 에러 아이콘 + "할 일을 불러올 수 없어요" + 재시도
- 완료 토글 실패: 토스트 + 체크 상태 롤백

## 5. 권한별 차이

| 요소 | ADMIN (주양육자) | USER (보조양육자) |
|------|------------------|-------------------|

## 6. API 연동

### 6-1. 데이터 needs (구체적으로 — API 계약의 입력)

**읽기(Read):**
| 데이터 | 필드 | 타입 | 비고 |
|--------|------|------|------|
| TODO 목록 | home_id | string | 홈 단위 |
| | status | enum(active/done) | 탭 분기 |
| | date_from / date_to | date | 완료 탭 필터, nullable |
| TODO 항목 | todo_id | string | |
| | title | string | 제목 |
| | due_date | date | 마감일 |
| | done_at | datetime | 완료 시각, nullable |
| | assignee | object(user_id,name) | 담당자 |
| | author | object(user_id,name) | 작성자 |
| 권한 판정 | current_user_id | string | 본인 담당/작성 여부 |
| | current_role | enum(ADMIN/USER) | 버튼·체크 분기 |

**쓰기(Write):**
| 액션 | 필드 | 타입 | 검증 | 비고 |

## 10. 접근성

| 항목 | 기준 |
|------|------|
| 터치 타깃 | 카드 72px, 체크박스 48px, 추가/FAB 48~56px |
| 색 대비 | 본문 #4f453f on #ffffff ≥ 4.5:1, 완료 취소선도 대비 유지 |
| 스크린리더 라벨 | 카드: "예방접종 예약, 6월 30일 마감, 담당 엄마, 진행 중", 체크박스: "완료로 표시", 비활성: "본인 담당이 아니어서 완료할 수 없음" |
