## 3. 상태 분기

### 기본 (데이터 있음)
- TODO 상세: 체크 상태(진행 중/완료), 기간·담당·작성·메모 표시
- 공지 상세: 제목·작성자·날짜·본문, 고정 시 핀 아이콘

### 빈 상태 (데이터 없음)
- 작성 폼 초기: 제목 비움, 기간 기본 오늘, 담당자 미선택(또는 본인)
- 상세 접근 불가(삭제됨): "삭제되었거나 찾을 수 없어요" + [목록으로]
- 공지 목록 없음(섹션): "아직 공지가 없어요" (ADMIN에게만 작성 CTA)

### 로딩 (스켈레톤)
- 상세: 헤더 + 카드 1~2장 shimmer (radius 20px)
- 저장/게시 중: 버튼 로딩 스피너, Optimistic 반영

### 에러
- 검증 실패: 인라인 에러, 저장 차단
- 저장/게시 실패: 토스트 + 재시도, Optimistic 롤백
- 권한 없음(USER가 공지 작성 시도): 진입 자체 차단(버튼 숨김), 우회 시 403 안내

## 5. 권한별 차이

| 요소 | ADMIN (주양육자) | USER (보조양육자) |
|------|------------------|-------------------|
| TODO 작성 | 가능 | 가능 |
| TODO 완료 체크 | 모든 TODO | 본인 담당만 (타인 담당 비활성) |
| TODO 수정 | 전체 | 본인 작성만 (타인 작성 읽기 전용) |
| TODO 삭제 | 가능 | 숨김 (정책상 USER 삭제 금지) |
| 공지 작성/게시 | 가능 | 숨김 (진입 불가) |
| 공지 수정·삭제(⋮) | 가능 | 숨김 |
| 공지 열람 | 가능 | 가능 (읽기 전용) |

> 정책(user_scenario 2-2, 11): 공지는 ADMIN만 작성·수정, USER 열람. TODO는 USER 작성·본인 담당 완료만, 삭제·타인 수정 불가.
