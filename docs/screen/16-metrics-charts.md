## 3. 상태 분기

### 기본 (데이터 있음)
- 차트 탭별 카드 표시, 선택 기간 범위로 시계열 플롯
- 성장 차트: 측정 데이터 라인 + WHO 표준곡선(정적) 오버레이
- 데이터 포인트 탭 시 값/날짜 툴팁
- 하단 요약 카드: 최근 측정값과 WHO 백분위

### 빈 상태 (측정 기록 없음)
- 각 차트 카드 자리에 `chart-line` 아이콘 + "아직 측정 기록이 없어요"
- WHO 표준곡선은 **참조용으로 항상 표시**(정적 데이터), 측정 라인만 비표시
- "성장 기록 추가하기" CTA → 기록 입력(성장) 진입

### 로딩 (스켈레톤)
- 기간 필터 chip shimmer + 차트 카드 3개 shimmer (radius 20px, h 240px)

### 에러
- 전체: 에러 아이콘 + "지표를 불러올 수 없어요" + 재시도
- 부분(특정 차트): 해당 카드만 재시도 버튼 (다른 차트는 정상 표시)

## 5. 권한별 차이

| 요소 | ADMIN (주양육자) | USER (보조양육자) |
|------|------------------|-------------------|
| 차트 조회 | 전체 | 전체 (동일) |
| 기간 필터 | 가능 | 가능 (동일) |
| WHO 오버레이 | 표시 | 표시 (동일) |
| 측정 기록 추가(요약 CTA) | 가능 | 가능 (생성은 공통) |
| 측정 기록 삭제 | 가능 | 불가 (정책상 USER 삭제 금지) |

> 지표 탭은 **조회 전용**이라 ADMIN/USER 차이가 없다. 데이터 생성/삭제는 기록 화면에서 처리.

## 6. API 연동

|--------|------|------|-----------|------|
| 공통 파라미터 | baby_id | string | 활성 아이 | 모든 차트 |
| | period | enum(`1w`/`1m`/`3m`/`6m`/`1y`) | 기본 `1m` | 기간 필터 |
| | from / to | date | from ≤ to, 미래 불가 | period로 산출 |

| | points[].age_months | number | ≥ 0 | WHO 매칭 키 |
| | unit | enum(`cm`/`kg`) | | 축 단위 표기 |

| | who.sex | enum(`male`/`female`) | baby.sex로 선택 | |

## 9. 데이터 예시

### 9-1. 초기 상태 (데이터 없을 때)
```
[지표] 성장 탭 · 기간 1개월

키 (cm)
  ─ WHO 곡선만 점선으로 표시 (참조 데이터)
  chart-line 아이콘
  "아직 측정 기록이 없어요"
  [성장 기록 추가하기] 버튼 → 기록(성장) 입력

(몸무게·머리둘레 카드 동일)

요약 카드
  "측정 기록을 추가하면 백분위를 보여드려요"
```

### 9-2. 더미 데이터 (정상 표시)
```
아이: 하은 (여아, 6개월) / 기간: 최근 3개월

[성장 탭]
키 (cm)        ● 측정   ─ WHO 50p
  60.1 (3개월) → 64.0 (4개월) → 66.5 (5개월) → 67.2 (6개월)
  WHO 50p 곡선 위/근접 → 백분위 50p

몸무게 (kg)    ● 측정   ─ WHO 50p
  5.9 → 6.8 → 7.4 → 7.8   (WHO 60p)

머리둘레 (cm)  ● 측정   ─ WHO 50p
  40.5 → 42.0 → 43.1 → 43.8

요약 카드
  키 67.2cm · 몸무게 7.8kg · 6/28 측정
  WHO 백분위: 키 50p · 몸무게 60p

[수유 탭] 기간 1주 · 일 집계
  6/24 720ml(8회) · 6/25 760ml(8회) · 6/26 800ml(9회)
  6/27 740ml(8회) · 6/28 780ml(8회) · 6/29 810ml(9회) · 6/30 690ml(7회)
  막대=수유량 / 라인=횟수

[수면 탭] 기간 1주 · 일 집계
  6/24 14h(낮 4h/밤 10h) · 6/25 13.5h · 6/26 15h · 6/27 14h
  6/28 13h · 6/29 14.5h · 6/30 14h
  막대=총 수면 / 영역=낮·밤 누적
```
