## 3. 상태 분기

### 기본 (데이터 있음)
- 프로필 카드에 사용자 정보·권한 pill 표시
- 권한에 따라 메뉴 노출/숨김 (5절 참조)

### 빈 상태 (데이터 없음)
- 해당 없음 (로그인 사용자 정보는 항상 존재). 아바타 이미지 없으면 이니셜 아바타 fallback

### 로딩 (스켈레톤)
- 프로필 카드 shimmer (h 88px, radius 20px) + 메뉴 카드 2그룹 shimmer (행 5 + 2)

### 에러
- 프로필 조회 실패: 프로필 카드 영역만 "내 정보를 불러올 수 없어요" + 재시도. 메뉴는 캐시/기본값으로 노출

## 5. 권한별 차이

| 요소 | ADMIN (주양육자) | USER (보조양육자) |
|------|------------------|-------------------|
| 권한 pill | "주양육자" (Primary tint) | "보조양육자" (Neutral) |
| 아이 관리 | 노출 (CRUD) | 노출 (조회·전환만, 추가/수정/삭제 숨김) |
| 구성원·초대 관리 | 노출 | **숨김** |
| 홈 관리 | 노출 | **숨김** |
| 기기 관리 | 노출 | **숨김** |
| 분유·젖병 설정 | 노출 | 노출 (조회+입력, 삭제 숨김) |
| 알림 설정 | 노출 | 노출 (동일) |
| 계정·회원 탈퇴 | 노출 | 노출 (동일) |

> USER에게 숨겨진 메뉴는 행 자체를 렌더링하지 않음 (disabled 처리 아님). "관리" 섹션에 아이 관리·분유젖병만 남을 수 있음.
