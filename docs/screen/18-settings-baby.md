## 3. 상태 분기

### 기본 (데이터 있음)
- 활성 아이 1건 + 다른 아이 N건. 활성은 상단 고정, 뱃지 표시
- 각 카드에 개월수/만나이 자동 계산 표기

### 빈 상태 (데이터 없음)
- 온보딩 직후 외에는 드묾. baby 아이콘 + "등록된 아이가 없어요" + "아이 추가하기" CTA (ADMIN). USER는 "주양육자가 아이를 등록하면 표시돼요"

### 로딩 (스켈레톤)
- 아이 카드 2행 shimmer (h 80px, radius 20px)

### 에러
- 전체: "아이 목록을 불러올 수 없어요" + 재시도
- 저장 실패: 바텀시트 유지 + 인라인 에러 토스트

## 4. 인터랙션

| 동작 | 대상 | 결과 |
|------|------|------|
| 탭 | 아이 카드 | 수정 바텀시트 (ADMIN) / 읽기 전용 상세 (USER) |
| 탭 | 전환하기 | baby_id 전환 → 전 화면 refetch, 설정 홈 복귀 (시나리오 12) |
| 탭 | 추가 버튼/추가 카드 | 추가 바텀시트 (ADMIN) |
| 탭 | 저장 | 검증 후 생성/수정, 목록 갱신 |
| 탭 | 아이 삭제 | 확인 다이얼로그 → 삭제 (ADMIN) |
| Pull-to-refresh | 전체 | 아이 목록 refetch |

## 5. 권한별 차이

| 요소 | ADMIN (주양육자) | USER (보조양육자) |
|------|------------------|-------------------|
| 아이 목록 조회 | 가능 | 가능 (동일) |
| 활성 아이 전환 | 가능 | 가능 |
| 추가 버튼/카드 | 노출 | **숨김** |
| 카드 탭 시 | 수정 바텀시트 | 읽기 전용 상세 (편집 필드 비활성) |
| 아이 삭제 | 가능 (확인 다이얼로그) | **숨김** |

## 6. API 연동

### 6-1. 데이터 needs

**읽기(Read):**
| 데이터 | 필드 | 타입 | 비고 |
|--------|------|------|------|
| 아이 목록 | home_id | string | 활성 홈 |
| 아이 항목 | baby_id | string | |
| | name | string | 1~20자 |
| | birth_date | date | 미래 불가 |
| | gender | enum(MALE/FEMALE) | |
| | avatar_url | string\|null | |
| | is_active | bool | 현재 선택 여부 |
| | age_months | int | 서버 또는 클라 계산 |

**쓰기(Write):**
| 액션 | 필드 | 타입 | 검증 | 비고 |
|------|------|------|------|------|
| 추가 | name | string | 필수, 1~20자 | ADMIN |
| | birth_date | date | 필수, ≤ 오늘 | |
| | gender | enum | 필수 | |
| 수정 | baby_id + 위 필드 | — | 부분 수정 | ADMIN |
| 삭제 | baby_id | string | ADMIN, 확인 필요 | 연관 데이터 영향 경고 |
| 활성 전환 | baby_id | string | — | 클라 상태 + 서버 last_active 갱신 |
