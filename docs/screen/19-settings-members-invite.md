## 3. 상태 분기

### 기본 (데이터 있음)
- 구성원 목록(역할 pill) + 초대 중 목록(만료 D-day)
- 본인 행에 "(나)" 표기, 역할 변경 불가

### 빈 상태 (데이터 없음)
- 구성원은 최소 1명(본인) 항상 존재. 초대 중 0건이면 "진행 중인 초대가 없어요" + "구성원 초대하기" CTA

### 로딩 (스켈레톤)
- 구성원 카드 2행 + 초대 카드 1행 shimmer

### 에러
- 전체: "구성원 정보를 불러올 수 없어요" + 재시도
- 초대 생성 실패(이메일 중복/이미 구성원): 바텀시트 인라인 에러
- 권한 변경 실패: 토스트 + 원복

## 5. 권한별 차이

| 요소 | ADMIN (주양육자) | USER (보조양육자) |
|------|------------------|-------------------|
| 화면 진입 | 가능 | **메뉴 숨김 (진입 불가)** |
| 구성원 목록 조회 | 가능 | (조회는 가능하나 메뉴 미노출 — 별도 진입점 없음) |
| 초대 생성 | 가능 | 불가 |
| 초대 취소 | 가능 | 불가 |
| 권한 변경 | 가능 (본인 제외) | 불가 |

> user_scenario 2-3: 초대/권한 변경은 ADMIN 전용. 설정 홈에서 USER에게 본 메뉴를 렌더링하지 않음.

## 7. 데이터·검증 규칙

| 입력 필드 | 필수 | 형식/범위 | 단위 | 에러 메시지 |
|-----------|------|-----------|------|-------------|
| invitee_email | Y | RFC 이메일 형식 | — | "올바른 이메일 형식이 아니에요" |
| invitee_email(중복) | — | 기존 구성원/PENDING 불가 | — | "이미 구성원이거나 초대 중인 이메일이에요" |
| role_id | Y | 1 또는 2 | — | "권한을 선택해주세요" |
| 권한 변경 | — | 마지막 ADMIN 강등 불가 | — | "주양육자가 최소 1명 있어야 해요" |
