## 3. 상태 분기

### 기본 (데이터 있음)
- 홈 요약(이름/구성원·아이 수/생성일) + 홈 정보 + 바로가기

### 빈 상태 (데이터 없음)
- 해당 없음 (홈은 항상 존재). 구성원/아이 0 시 카운트만 "0명" 표기

### 로딩 (스켈레톤)
- 요약 카드 + 정보 카드 2행 shimmer

### 에러
- 전체: "홈 정보를 불러올 수 없어요" + 재시도
- 이름 저장 실패: 시트 인라인 에러 + 원복

## 5. 권한별 차이

| 요소 | ADMIN (주양육자) | USER (보조양육자) |
|------|------------------|-------------------|
| 화면 진입 | 가능 | **메뉴 숨김 (진입 불가)** |
| 홈 이름 수정 | 가능 | 불가 |
| 홈 ID 복사 | 가능 | (진입 불가) |
| 바로가기 | 가능 | (진입 불가) |

> user_scenario 2-3: 홈 설정은 ADMIN 전용. 설정 홈에서 USER에게 본 메뉴 미렌더링.

## 6. API 연동

### 6-1. 데이터 needs

**읽기(Read):**
| 데이터 | 필드 | 타입 | 비고 |
|--------|------|------|------|
| 홈 정보 | home_id | string | 복사용 |
| | name | string | 1~20자 |
| | member_count | int | 요약 |
| | baby_count | int | 요약 |
| | created_at | date | 생성일 |

**쓰기(Write):**
| 액션 | 필드 | 타입 | 검증 | 비고 |
|------|------|------|------|------|
| 이름 변경 | name | string | 필수, 1~20자, trim | ADMIN |

### 6-2. 엔드포인트 매핑 (TBD)

| 액션 | METHOD /path | 비고 |
|------|--------------|------|
| 홈 조회 | `TBD` (GET /homes/{id}) | 카운트 포함 |
| 이름 변경 | `TBD` (PATCH /homes/{id}) | USER 403 |
