## 3. 상태 분기

### 기본 (데이터 있음)
- 젖병: context 확정 카드 + 측정값 + 집계 칩(현재 방식 선택)
- 분유: ACTIVE 1건(상단) + 이전 분유 N건

### 빈 상태 (데이터 없음)
- 젖병: "아직 측정값이 없어요. 빈 젖병 무게를 3회 이상 측정해주세요" + "측정 추가" / context 카드는 "미확정 — 자동출수 비활성" 배너
- 분유: baby-bottle 아이콘 + "등록된 분유가 없어요" + "분유 등록"

### 자동출수 리다이렉트 진입 (젖병 미등록)
- 상단에 인라인 배너 표시: "⚠️ 이 기능을 사용하려면 젖병 등록이 필요해요"
- 젖병 탭 자동 포커스, 측정 추가 CTA 강조(Primary)
- 등록 완료 시 배너 사라지고 이전 화면(자동출수)으로 자동 복귀

### 기기 등록 후 연속 진입
- 상단에 안내 배너: "기기 등록이 완료됐어요. 젖병과 분유를 등록하면 자동 출수를 시작할 수 있어요"
- 젖병 탭부터 순서대로 안내 (젖병 캘리브레이션 → 분유 등록)

### 로딩 (스켈레톤)
- 각 탭 카드 2행 shimmer

### 에러
- 전체: "정보를 불러올 수 없어요" + 재시도
- 확정 실패: 토스트 + 칩 선택 유지
- 분유 등록/교체 실패: 시트 인라인 에러
- ACTIVE 삭제 시도: 버튼 비활성 + "사용 중인 분유는 삭제할 수 없어요" 툴팁

## 4. 인터랙션

| 동작 | 대상 | 결과 |
|------|------|------|
| 탭 | 세그먼트(젖병/분유) | 탭 전환 |
| 탭 | 측정 추가 | 측정값 입력 시트 → 칩 추가 |
| 롱프레스 | 측정값 칩 | 삭제 확인 (오입력 제거) |
| 탭 | 집계 방식 칩 | 대표 무게 미리보기 갱신 |
| 탭 | 대표 무게 확정 | dispense_context 저장 → "확정" + 자동출수 활성 |
| 탭 | 분유 등록 | 등록 시트 (종류/g/ml) |
| 탭 | 교체 | 교체 시트 → TRANSITION 생성, 기존 ACTIVE→INACTIVE |
| 탭 | 삭제(🗑) | INACTIVE만, 확인 다이얼로그 (ADMIN). ACTIVE 비활성 |
| Pull-to-refresh | 전체 | 젖병·분유 refetch |

## 5. 권한별 차이

| 요소 | ADMIN (주양육자) | USER (보조양육자) |
|------|------------------|-------------------|
| 화면 진입/조회 | 가능 | 가능 |
| 측정값 추가 | 가능 | 가능 (입력 허용) |
| 대표 무게 확정 | 가능 | 가능 (입력 허용) |
| 분유 등록 | 가능 | 가능 |
| 분유 교체(TRANSITION) | 가능 | 가능 (입력 범주) |
| 측정값 칩 삭제 | 가능 | **숨김** |
| 분유 삭제(INACTIVE) | 가능 | **숨김** |
| ACTIVE 분유 삭제 | **불가 (정책상 금지)** | 불가 |

> user_scenario 2-2: 분유·젖병은 USER도 조회+입력 가능, 삭제만 ADMIN. ACTIVE 분유는 누구도 삭제 불가(정책).

## 10. 접근성

| 항목 | 기준 |
|------|------|
| 터치 타깃 | 카드 80px, 집계 칩 44px+, 삭제/교체 버튼 44px+ |
| 색 대비 | 본문 #4f453f on #ffffff ≥ 4.5:1, 상태 뱃지 색+텍스트 병기 |
| 스크린리더 라벨 | context: "대표 무게 72그램, 집계 방식 중앙값, 확정됨", 분유: "압타밀 1단계, 사용 중, 스쿱당 4.3그램 30밀리리터" |
| 포커스 순서 | 세그먼트 탭 → context 카드 → 측정값 → 집계 칩 → 확정 / 분유: ACTIVE → 이전 → 등록 |
