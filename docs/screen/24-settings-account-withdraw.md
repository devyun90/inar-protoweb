## 3. 상태 분기

### 기본 (데이터 있음)
- 계정 정보(이메일/로그인 방식/가입일) + 로그아웃 + 회원 탈퇴
- 탈퇴 플로우 3단계(안내→사유→최종 확인)

### 빈 상태 (데이터 없음)
- 해당 없음 (로그인 사용자 정보는 항상 존재)

### 로딩 (스켈레톤)
- 계정 정보 카드 shimmer / 탈퇴 처리 중 전체 오버레이 스피너 + "탈퇴 처리 중이에요"

### 에러
- 계정 조회 실패: "계정 정보를 불러올 수 없어요" + 재시도
- 탈퇴 실패: "탈퇴를 처리하지 못했어요. 잠시 후 다시 시도해주세요" + 재시도 (Step3 유지)
- ADMIN 단독 경고는 서버의 role/홈 구성 조회 결과로 분기 (조회 실패 시 안전하게 경고 표시)

## 4. 인터랙션

| 동작 | 대상 | 결과 |
|------|------|------|
| 탭 | 로그아웃 | 확인 다이얼로그 → 세션 종료 → 로그인 화면 |
| 탭 | 회원 탈퇴 | 탈퇴 안내(Step1) 진입 |
| 탭 | 계속(Step1) | 사유(Step2)로 이동 |
| 탭 | 취소(Step1/2/3) | 계정 화면으로 복귀 (플로우 중단) |
| 선택 | 탈퇴 사유 | 라디오 선택, 기타 시 입력 활성 |
| 탭 | 다음(Step2) | 최종 확인(Step3)으로 이동 |
| 입력 | 확인 텍스트 | "탈퇴합니다" 정확 일치 시 탈퇴 버튼 활성화 |
| 탭 | 탈퇴합니다(Step3) | 탈퇴 API → 성공 시 로그인 화면 |

## 5. 권한별 차이

| 요소 | ADMIN (주양육자) | USER (보조양육자) |
|------|------------------|-------------------|
| 화면 진입/조회 | 가능 | 가능 |
| 로그아웃 | 가능 | 가능 |
| 회원 탈퇴 | 가능 | 가능 |
| ADMIN 단독 경고 | **단독 ADMIN일 때 홈 삭제 경고 표시** | 미표시 (본인 탈퇴는 구성원 이탈) |
| 탈퇴 영향 | 단독 시 홈·아이 전체 삭제 / 복수 ADMIN 시 본인 데이터만 | 본인 계정·작성 데이터 처리, 홈 유지 |

> user_scenario 15: ADMIN 단독 시 홈·아이 데이터 전체 삭제 경고 필수. 다른 ADMIN/구성원이 있으면 홈은 유지.

## 6. API 연동

### 6-1. 데이터 needs

**읽기(Read):**
| 데이터 | 필드 | 타입 | 비고 |
|--------|------|------|------|
| 계정 정보 | email | string | |
| | auth_provider | enum(GOOGLE/APPLE) | 로그인 방식 |
| | created_at | date | 가입일 |
| 탈퇴 영향 | is_sole_admin | bool | ADMIN 단독 여부 → 경고 분기 |
| | home_will_be_deleted | bool | 홈 삭제 여부 |
| | retention_days | int | 보관 기간(30) |

**쓰기(Write):**
| 액션 | 필드 | 타입 | 검증 | 비고 |
|------|------|------|------|------|
| 로그아웃 | — | — | — | 세션/토큰 폐기 |
| 회원 탈퇴 | reason_code | enum\|null | 선택 | UNUSED/LACK_FEATURE/BUGS/ETC |
| | reason_text | string\|null | 기타 시 1~200자 | |
| | confirm_phrase | string | "탈퇴합니다" 정확 일치 | 클라 검증 + 서버 재확인 |

### 6-2. 엔드포인트 매핑 (TBD)

|------|--------------|------|
