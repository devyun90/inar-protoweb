## 3. 상태 분기

### 기본 (플레이스홀더 — MVP 상시 상태)
- 항상 동일한 "곧 제공됩니다" 화면 표시 (데이터 호출 없음)
- 준비 중 기능 카드는 흐림/잠금 처리(상호작용 비활성)

### 빈 상태 (데이터 없음)
- 플레이스홀더와 동일 (별도 빈 상태 없음 — 데이터 개념 자체가 없음)

### 로딩 (스켈레톤)
- 해당 없음 (네트워크 호출 없이 즉시 렌더)
- "알림 받기" 등록 시에만 버튼 로딩 스피너

### 에러
- 화면 자체 에러 없음
- "알림 받기" 등록 실패 시: 토스트 "잠시 후 다시 시도해주세요" + 재시도
