# 앨범 탭 — 준비 중 (플레이스홀더)

> 시나리오: user_scenario 시나리오 10 (앨범 & 일기) · 범위: Phase 2 (MVP는 플레이스홀더)
> 연계: [user_scenario.md](../user_scenario.md) · [screen_scenario.md](../screen_scenario.md) · [design rc.4](../1/design.md)

| 항목 | 내용 |
|------|------|
| 위치 | 앨범 탭 (Bottom Tab 3번째) |
| 역할(권한) | 공통 (조회·기능 없음, ADMIN/USER 동일) |
| 연결 화면 | [홈/지표 탭] → **[앨범 탭(준비 중)]** → (이동 없음, 탭 복귀) |
| 디바이스 | MOBILE (390px 기준, Safe Area 상/하) |

> **정책:** 앨범 탭은 **숨기지 않는다.** MVP에서도 Bottom Tab에 노출하되 진입 시 "준비 중" 플레이스홀더를 표시한다. Phase 2 런칭 시 사용자 학습 비용을 최소화하기 위함.

## 3. 상태 분기

### 기본 (데이터 있음)
- 해당 없음 — 항상 동일한 "준비 중" 플레이스홀더 표시 (데이터 페치 없음)

### 빈 상태 (데이터 없음)
- 기본 상태와 동일 (이 화면 자체가 안내 화면)

### 로딩 (스켈레톤)
- 없음 — 정적 화면, 네트워크 호출 없음 (즉시 렌더)

### 에러
- 없음 — API 호출이 없어 에러 분기 불필요. (Phase 2에서 실제 데이터 연동 시 추가)
